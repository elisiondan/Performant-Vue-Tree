# Custom node properties

In section basic usage was outlined what properties each node in the tree must contain. However, feel are free to define as many custom properties you need. Since you may want to access nodes in various scenarios, for example through scoped slots or through tree traversal, it can be beneficial to do so.

For instance, imagine that you would like to define a custom icon for nodes. You could define the icon name at initialization and then resolve it through scoped slots. It might be a good practice to prefix your custom properties (with an underscore for example).

// data definition
treeData: [
        {
            id: 'root1',
            name: 'Root,
            _icon: 'folder.svg'
            children: [{
                id: 'child1',
                name: 'Child',
                _icon: 'file.svg',
                children: []
            }],
        }
    ]
}

<performant-vue-tree :data="treeData">
    <template #nodeIcon="{nodeData}">
        <img :src=`/icons/${nodeData._icon}` />
    </template>
</performant-vue-tree>
